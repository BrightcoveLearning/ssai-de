    Clientseitige Pre-Roll-Anzeigen mit Live SSAI

    In diesem Thema erfahren Sie, wie Sie Brightcove Player verwenden, um eine clientseitige Pre-Roll-Anzeige mit Live-Streams abzuspielen, die für serverseitige Anzeigeneinfügung (SSAI) aktiviert sind.

    Übersicht

    Wenn Sie Live-Streams mit serverseitigem Anzeigeneinfügung (SSAI) mit Brightcove Player bereitstellen, können Sie eine clientseitige Pre-Roll-Anzeige einfügen. Bei clientseitigen Anzeigen unterstützt diese Funktion IMA-Anzeigen.

    Für diese Funktion ist Folgendes erforderlich:

    • IMA3-Plugin 3.5.0+
    • SSAI-Plugin 1.11.0+
    • Brightcove Player Version 6.44.0+

    Vorbehalte

    • Kunden, die diese Funktion verwenden, müssen Dynamic Delivery verwenden.
    • Es werden nur IMA- und SSAI-Plugins unterstützt (kein FreeWheel).
    • Derzeit unterstützt diese Funktion nur einen einzelnen Spieler auf der Seite.

    Erste Schritte

    Gehen Sie folgendermaßen vor, um eine clientseitige IMA-Porroll-Anzeige mit einem Live-SSAI-Stream abzuspielen:

    1. Einen SSAI-fähigen Livestream erstellen
    2. Erstelle einen Brightcove Spieler
    3. Implementieren der clientseitigen Preroll mit Studio

    Einen SSAI-fähigen Livestream erstellen

    Das Live-Modul unterstützt serverseitige Anzeigeneinfügung (SSAI), sodass serverseitige Anzeigen während eines Live-Streams angefordert und angezeigt werden können. Informationen zum Erstellen Ihres Live-Streams finden Sie in den folgenden Themen:

    • Implementierung von serverseitigen Anzeigen im Live-Modul

    Erstellen eines Brightcove-Players

    Erstellen Sie einen neuen Brightcove Player mit Video Cloud Studio. Einzelheiten finden Sie im Schnellstart: Erstellen und Styling eines Player-Dokuments

    Sie müssen einen Brightcove Player Version 6.44.0+ verwenden.

    Implementieren der clientseitigen Preroll mit Studio

    Der einfachste Weg, Ihren Player für Auto-Failover-Anzeigen zu konfigurieren, ist Video Cloud Studio. Sobald Sie eine Anzeigenkonfiguration und einen Player erstellt haben, können Sie den Player wie folgt für das automatische Failover konfigurieren:

    1. Öffnen Sie das PLAYERS-Modul und suchen Sie den Player, dem Sie Werbefunktionen hinzufügen möchten.
    2. Wählen Sie den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.
    3. Wählen Sie im linken Navigationsmenü Werbungaus.
    4. Aktivieren Sie das Kontrollkästchen Clientseitig aktivieren (IMA) .

    5. Geben Sie die URL für Ihr IMA-Anzeigen-Tag an. In diesem Beispiel verwenden wir die Ad Tag-Beispiel-URL.

      Einzelheiten zu den Eigenschaften der Player-Werbung finden Sie im Dokument Konfigurieren von Spielerwerbung mit dem Players Module .

    6. Aktivieren Sie das Kontrollkästchen Serverseitig (SSAI) aktivieren.

      Die serverseitige Option wird nur angezeigt, wenn Ihr Konto für SSAI aktiviert wurde.
    7. Wählen Sie im Dropdown-Menü Konfiguration auswählen die Anzeigenkonfiguration aus, die Sie diesem Player zuordnen möchten.

    8. Wenn Sie möchten, dass Overlays über Ihre Anzeigen geschaltet werden, aktivieren Sie das Kontrollkästchen Anzeigeninformations-Overlays aktivieren. Dies umfasst „Mehr erfahren“ und Überlagerungen bei der Anzeigenabzählung.

      Overlays werden erst angezeigt, wenn Sie sie hier aktivieren und in Ihrem VAST Ad-Tag konfigurieren.
    9. Wählen Sie die Schaltfläche „ Speichern “.

    10. Wählen Sie im linken Navigationsmenü JSON Editoraus.

    11. Scrollen Sie im JSON-Editor nach unten, bis Sie die ad_failover: true Eigenschaft sehen.

    12. Ersetzen Sie die ad_failover: true Eigenschaft durch Folgendes: 

      "ima_preroll_with_ssai": true

    13. Ihr JSON-Editor sollte ähnlich aussehen:

    14. Wählen Sie die Schaltfläche „ Speichern “.

    15. Um den Player zu veröffentlichen, wählen Sie Veröffentlichungs- und Einbettungsänderungen einbetten >aus.

    16. Jetzt sind Sie bereit, Ihr Live-Event zu veröffentlichen. Einzelheiten finden Sie im Dokument Implementieren von serverseitigen Anzeigen im Live Module .
