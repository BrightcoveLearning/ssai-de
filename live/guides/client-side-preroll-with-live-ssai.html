Überblick

Wenn Sie mit Brightcove Player Live-Streams mit serverseitiger Anzeigeneinfügung (SSAI) bereitstellen, können Sie eine clientseitige Pre-Roll-Anzeige einfügen. Bei clientseitigen Anzeigen unterstützt diese Funktion IMA-Anzeigen.

Diese Funktion erfordert Folgendes:

  • IMA3 Plugin v3.5.0 +
  • SSAI Plugin v1.11.0 +
  • Brightcove Player Version 6.44.0+

Vorsichtsmaßnahmen

  • Kunden, die diese Funktion verwenden, müssen Dynamic Delivery verwenden.
  • Es werden nur IMA- und SSAI-Plugins unterstützt (kein FreeWheel).
  • Derzeit unterstützt diese Funktion nur einen einzelnen Player auf der Seite.

Einstieg

Gehen Sie folgendermaßen vor, um eine clientseitige IMA-Pre-Roll-Anzeige mit einem Live-SSAI-Stream abzuspielen:

  1. Erstellen Sie einen SSAI-fähigen Live-Stream
  2. Erstellen Sie einen Brightcove-Player
  3. Implementieren Sie clientseitiges Pre-Roll mit Studio

Erstellen Sie einen SSAI-fähigen Live-Stream

Das Live-Modul unterstützt SSAI (Server-Side Ad Insertion), sodass serverseitige Anzeigen während eines Live-Streams angefordert und angezeigt werden können. Informationen zum Erstellen Ihres Live-Streams finden Sie unter:

  • Implementieren von serverseitigen Anzeigen im Live-Modul

Erstellen Sie einen Brightcove-Player

Erstellen Sie mit Video Cloud Studio einen neuen Brightcove-Player. Einzelheiten finden Sie in der Schnellstart: Erstellen und Gestalten eines Players Dokument.

Sie müssen einen Brightcove Player Version 6.44.0+ verwenden.

Implementieren des clientseitigen Pre-Rolls mit Studio

Die einfachste Möglichkeit, Ihren Player für Auto-Failover-Anzeigen zu konfigurieren, ist Video Cloud Studio. Nachdem Sie eine Anzeigenkonfiguration und einen Player erstellt haben, können Sie den Player wie folgt für das automatische Failover konfigurieren:

  1. Öffne das SPIELER Modul und suchen Sie den Player, dem Sie Werbefunktionen hinzufügen möchten.
  2. Wählen Sie den Link für den Spieler, um die Eigenschaften des Spielers zu öffnen.
  3. Wählen Sie im linken Navigationsmenü Werbung.
  4. Überprüf den Aktivieren Sie clientseitig (IMA) Kontrollkästchen.

  5. Geben Sie die URL für Ihr IMA-Anzeigen-Tag an. In diesem Beispiel verwenden wir die Beispiel-URL für Anzeigen-Tags.

    Einzelheiten zu den Werbeeigenschaften des Players finden Sie in der Konfigurieren der Player-Werbung mithilfe des Players-Moduls Dokument.

  6. Überprüf den Aktivieren Sie die serverseitige (SSAI) Kontrollkästchen.

    Die serverseitige Option wird nur angezeigt, wenn Ihr Konto für SSAI aktiviert wurde.
  7. Von dem Wählen Sie Konfiguration Wählen Sie im Dropdown-Menü die Anzeigenkonfiguration aus, die Sie diesem Player zuordnen möchten.

  8. Wenn Sie möchten, dass Overlays über Ihren Anzeigen angezeigt werden, aktivieren Sie das Kontrollkästchen Aktivieren Sie Anzeigeninformations-Overlays Kontrollkästchen. Dies beinhaltet "Weitere Informationen" und Anzeigen-Countdown-Overlays.

    Overlays werden erst angezeigt, wenn Sie sie hier aktiviert und in Ihrem VAST Ad-Tag konfiguriert haben.
  9. Wähle aus speichern Taste.

  10. Wählen Sie im linken Navigationsmenü JSON-Editor.

  11. Scrollen Sie im JSON-Editor nach unten, bis Sie das sehen ad_failover: true Eigentum.

  12. Ersetze das ad_failover: true Eigenschaft mit dem folgenden: 

    "ima_preroll_with_ssai": true

  14. Wähle aus speichern Taste.

  15. Um den Player zu veröffentlichen, wählen Sie Veröffentlichen & Einbetten> Änderungen veröffentlichen.

  16. Jetzt können Sie Ihr Live-Event veröffentlichen. Einzelheiten finden Sie in der Implementieren von serverseitigen Anzeigen im Live-Modul Dokument.
