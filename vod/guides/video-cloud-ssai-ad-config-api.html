Anzeigenkonfigurationen definieren verschiedene Aspekte der SSAI-Wiedergabe, einschließlich Anzeigenaufrufen, Beacons und anderen Konfigurationsoptionen. Ein Konto kann mehrere Konfigurationen haben und derzeit können Konfigurationen nicht für mehrere Konten freigegeben werden.

Sie können Anzeigenkonfigurationen ganz einfach in Video Cloud Studio erstellen, auflisten und löschen. Einzelheiten finden Sie im Serverseitige Anzeigeneinstellungen konfigurieren dokumentieren.
Anmerkungen:
  • Um SSAI verwenden zu können, muss Ihr Video Cloud-Konto konfiguriert sein für Dynamische Lieferung und für SSAI aktiviert. Wenden Sie sich an Ihren Account Manager, um diese Funktion zu nutzen.
  • SSAI arbeitet mit Video-on-Demand (VOD) und unterstützt die Live-Wiedergabe.
  • Eine Liste der Einschränkungen finden Sie in der SSAI-Übersicht Dokument.

Allgemeine Information

Die folgenden Informationen beziehen sich auf alle SSAI-API-Anforderungen.

Basis-URL

Die Basis-URL für die SSAI-API lautet:

https://ssai.api.brightcove.com/v1

Kontopfad

In allen Fällen werden Anfragen für ein bestimmtes Video Cloud-Konto gestellt. Sie werden also immer den Begriff hinzufügen accounts gefolgt von Ihrer Konto-ID zur Basis-URL:

https://ssai.api.brightcove.com/v1/accounts/your account id

Authentifizierung

Die Authentifizierung für Anforderungen erfordert einen Autorisierungsheader:

Authorization: Bearer your access token

Die access_token ist ein temporäres OAuth2-Zugriffstoken, das von der Brightcove-OAuth Service. Ausführliche Informationen zum Abrufen von Client-Anmeldeinformationen und zum Abrufen von Zugriffstoken finden Sie unter Brightcove OAuth Übersicht.

Operationen

Wenn Sie Client-Anmeldeinformationen anfordern, müssen Sie die Art des Kontozugriffs oder der gewünschten Vorgänge angeben. Im Folgenden finden Sie eine Liste der derzeit unterstützten Vorgänge für die SSAI-API:

  • SSAI-Daten:

    video-cloud/ssai/read
    video-cloud/ssai/all

Die API unterstützt die folgenden GET- und PUT-Anforderungen:

  • Anzeigenkonfigurationen auflisten

Details zum Konfigurationsfeld

In dieser Tabelle werden die Anzeigenkonfigurationsfelder definiert, die im Hauptteil einer Anforderung verwendet werden.

Feld Typ Beschreibung
name String Vom Menschen lesbarer Name
vmap_response_namespace String Passt die VMAP-Ausgabe an, um das alte Unicorn Once-Namespace-Format oder das neue Brightcove-Namespace-Format zu verwenden.

Werte: uo, bc Default: bc
description String Vom Menschen lesbare Beschreibung
ad_config.expected_ad_response String Welche Technologie zum Analysieren der Antwort verwendet werden soll. [1]

Werte:
  • dfp_ad_rules
  • dfp_vmap
  • smart_xml
  • vast_3_0
ad_config.disable_server_beacons Boolescher Wert Gibt an, ob das serverseitige Auslösen von Anzeigenimpressionen/Beacons deaktiviert werden soll

Wenn diese Option auf eingestellt ist true, löst SSAI keine Beacons serverseitig aus und schließt alle Beacons in die VMAP-Ausgabe ein

Wenn diese Option auf eingestellt ist false, wird SSAI die Beacons auslösen, die serverseitig sind, und alle, die es nicht kann, in die VMAP-Ausgabe aufnehmen
ad_config.round_up_cue_points Boolescher Wert Markiert, ob bei Auswahl einer Position in der Nähe zum Einfügen von Anzeigen auf den nächsten Keyframe gerundet werden soll.

Standard: false, der den nächstgelegenen Keyframe zur gewünschten Anzeigenposition auswählt.
ad_config.template_url.template String Anzeigen-Tag-Vorlage. Verfügbare Variablen, die in einem späteren Abschnitt beschrieben werden.
ad_config.template_url.defaults Objekt Zuordnung von String zu String, der den Standardwert definiert, der verwendet werden soll, wenn kein URL-Parameter angegeben wird.

Kann wörtlich sein { "url.foo": "SomeString" }

Oder verweisen Sie auf eine andere Variable {% raw %}{ "url.foo": "{{ metadata.title_id }}" }{% endraw %}
discontinuities.dash [2] [] String Steuert, welche Versionen von Dash für die Bereitstellung von Multi Period Dash-Manifesten verwendet werden sollen.

Einstellen ["*"] um Mehrperioden-Dash für alle Versionen zu liefern

Leere Liste für nie

Beispiel: ["live-timeline"] für Live-Timeline liefern, aber nicht für hbbtv
discontinuities.hls [2] [] String Steuert, welche Versionen von hls mit Diskontinuitäten geliefert werden sollen.

Einstellen ["*"] bis zur Auslieferung mit Unterbrechungen in allen Versionen von HLS

Leere Liste für nie

Beispiel: ["v4","v5"] für v4 & v5 zu liefern, aber nicht für v3
beacon_templates Array Eine Reihe von Beacons, die ausgelöst werden sollen (Beispiel: Beacons von Drittanbietern)
beacon_templates[].type String

Art des zu feuernden Leuchtfeuers. Werte:

  • content_start
  • content_first_quartile
  • content_midpoint
  • content_third_quartile
  • content_complete
  • content_quartiles
  • content_interval
  • ad_start
  • ad_first_quartile
  • ad_midpoint
  • ad_third_quartile
  • ad_complete
  • ad_quartiles
  • ad_break_start
  • ad_break_end
  • segment_start
  • segment_end
  • on_load
beacon_templates[].template_url.template String Beacon-URL-Vorlage
extend_beacon_guard_ttl Boolescher Wert Legt die Länge der Beacon Guard-TTL (Time to Live) auf die Länge der Session-TTL des Inhalts fest. Andernfalls beträgt die Standardeinstellung 1 Minute.

Die folgenden Hinweise beziehen sich auf die Felder in der obigen Tabelle.

[1] Wenn der auf eingestellt expected_ad_response ist vast_3_0, führt SSAI für jeden in Video Cloud definierten Anzeigen-Cue-Point einen VAST Cue Point durch. Für VMAP- und Anzeigenregeln stellt SSAI Anforderungen basierend auf den definierten Anzeigenunterbrechungen in der ersten Anzeigenantwort.

[2] Diskontinuitäts-Tags weisen auf eine Änderung der Frameraten-Codierung zwischen Content-Video- und Anzeigensegmenten hin. Sie weisen das System an, die Rendering-Kette zu rekonstruieren. Für skalierbare SSAI muss der Videoplayer HLS mit Discontinuity-Tags und MPEG DASH mit mehreren Perioden abspielen können.

Anzeigenvariablen

Variablen in der Vorlagen-URL werden durch doppelte geschweifte Klammern ({% raw %}{{ … }}{% endraw %}) mit optionalem Leerzeichen vor und nach dem variablen Pfad identifiziert. Allen Variablen wird einer von drei Namespaces vorangestellt:

  • System
  • url
  • Metadaten

Systemvariablen

Systemvariablen werden vom SSAI-System bereitgestellt und können Informationen über den Endbenutzer oder Hilfsvariablen sein, um Zufallswerte zu generieren. Die Werte müssen URI-codiert sein, bevor sie in die URL-Vorlagen eingefügt werden.

Systemvariablen werden identifiziert als: {% raw %}{{system.*}}{% endraw %}

Feld Beschreibung
ip_address IP-Adresse des Endbenutzers
random_number_32 Zufällige 32-Bit-Ganzzahl
random_guid Zufällige UUID
referer Referer-Header-Wert des Endbenutzers
timestamp_utc Aktuelle Zeit als Unix-Zeitstempel
unique_user_id MD5 (IP-Adresse + Benutzeragent)
unix_timestamp Aktuelle Zeit als Unix-Zeitstempel (Sekunden)
user_agent User-Agent-Header-Wert des Endbenutzers
uuid Zufällige UUID
x_forwarded_for X-Forwarded-For-Header-Wert des Endbenutzers
xfp.correlator Zufällige 64-Bit-Ganzzahl
xfp.ip_address IP-Adresse des Endbenutzers
xfp.unique_user_id MD5 (IP-Adresse + Benutzeragent)
xfp.scor Zufällige 64-Bit-Ganzzahl

URL-Variablen

Abfrageparameter, die auf dem Einstiegspunkt VMAP / Manifest bereitgestellt werden, sind unter verfügbar url Namespace. Im Gegensatz zu Variablen unter anderen Namespaces werden diese Parameter beim Einfügen in die Vorlage nicht als URL codiert. Wenn Variablenwerte an den Anzeigenanbieter URL-codiert werden müssen, müssen sie auf der Einstiegspunkt-URL doppelt URL-codiert sein.

URL-Variablen werden wie folgt identifiziert: {% raw %}{{url.*}}{% endraw %}

Metadatenvariablen

Metadatenvariablen beschreiben das Inhaltsvideo, das sowohl aus Video Cloud- als auch aus Dynamic Delivery-Datenquellen stammt. Die Werte werden URL-codiert, bevor sie in die URL-Vorlagen eingefügt werden.

Metadatenvariablen werden wie folgt identifiziert: {% raw %}{{metadata.*}}{% endraw %}

Feld Beschreibung
ad_keys Freiform-Textzeichenfolge, die im Video Cloud Studio Media-Modul hinzugefügt und bearbeitet werden kann
{% raw %}custom_fields.{field_name}{% endraw %} Benutzerdefinierte Video Cloud-Felder
long_description Video Cloud lange Beschreibung
name Video Cloud Videoname
reference_id Video Cloud Referenz-ID
tags Durch Kommas getrennte Liste der Video Cloud-Tags für das Video
title.duration Dauer des Inhalts in Sekunden
title.id Titel-ID für dynamische Zustellung
title.name Name des Dynamic Delivery-Titels
video_id Video Cloud Video-ID

Einstiegspunkt-URL-Parameter

Es gibt eine Handvoll Abfrageparameter, die der SSAI-Einstiegspunkt-URL (VMAP oder Manifest) hinzugefügt werden können, um einige Verhaltensweisen zu optimieren:

Parameter Beschreibung
?rule=sd-only Filtert alle Videowiedergaben heraus, deren Höhe unter dem in der Kontokonfiguration festgelegten Schwellenwert liegt
?rule=discos-enabled Aktivieren Sie die Wiedergabe mit Diskontinuitäten in HLS und MultiPeriods in Dash. Hat Vorrang vor der Einstellung der Diskontinuitäten in der Wiedergabekonfiguration
?rule=discos-disabled Deaktivieren Sie die Wiedergabe mit Diskontinuitäten in HLS und MultiPeriods in Dash. Hat Vorrang vor der Einstellung der Diskontinuitäten in der Wiedergabekonfiguration

Konfigurationshinweise

  1. Das Vorladen von Anzeigen sollte nicht mit SSAI erfolgen. Der Grund dafür ist, dass der Player beim Vorladen eine Anzeigenimpression und wahrscheinlich die ersten Quartil-Beacons meldet, bevor das Video abgespielt wird. Dies kann zu ungenauen Anzeigenanalysen führen. Wenn Sie SSAI in Studio konfigurieren, wird dies automatisch durchgeführt. Wenn Sie SSAI jedoch manuell einrichten, müssen Sie sich dieses Problems bewusst sein.
  2. Wenn der Web-Player SSAI verwendet und eine Ihrer Beweggründe darin besteht, Werbeblocker zu umgehen, sollten Sie serverseitige Beacons verwenden. Clientseitige Beacons sollten nicht verwendet werden, da sie blockiert werden.

Clientseitige Makros

Verwenden Sie die page Präfix, wenn Sie clientseitige Anzeigenmakros verwenden möchten. Mit diesen Makros können Sie Variablen in den VMAP- und Server-URLs verwenden. Einzelheiten zu Anzeigenmakros finden Sie in der Anzeigenmakros und die serverURL Abschnitt der Werbung mit dem IMA3-Plugin-Dokument.

Clientseitigen Makros wird Folgendes vorangestellt: {% raw %}{{page.*}}{% endraw %}

Zum Beispiel, um die hinzuzufügen document.referrer und ein pageVariable DOM-Fenstervariable, die Sie ihnen voranstellen würden page in der Anzeigenvorlage wie folgt:

{% raw %}https://adserver.com/{{page.document.referrer}}/{{page.pageVariable.whateverIwant}}{% endraw %}

Die Wiedergabe-API kehrt zurück document.referrer und pageVariable.whateverIwant an die VMAP- und SRC-URLs angehängt. Der Brightcove-Player durchläuft dann seine clientseitige Makroersetzungslogik, um die entsprechenden Werte zu ersetzen, bevor die Anforderung gesendet wird:

{% raw %}https://bolt-prefix/blah.vmap?document.referrer={document.referrer}&pageVariable.whateverIwant={pageVariable.whateverIwant}{% endraw %}

Anzeigenfehler-Beacons

VAST Ad Error Beacons bei Verwendung von SSAI können hilfreich sein, um Probleme mit Ihrem Anzeigenworkflow proaktiv zu finden und zu beheben. Einzelheiten finden Sie in der Anzeigenfehler-Beacons mit SSAI Dokument.
