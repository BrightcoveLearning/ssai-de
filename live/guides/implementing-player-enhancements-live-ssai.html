    Implementieren von Spieler-Erweiterungen für Live-SSAI

    In diesem Thema erfahren Sie, wie Sie einen Live-Stream mit serverseitigen Anzeigen erstellen, der Player-Enhancements for Live SSAI für eine verbesserte Benutzererfahrung verwendet.

    Einführung

    Die Player-Enhancements für Live SSAI von Brightcove ermöglicht es Ihnen, die Werbeunterbrechungen in Live-Streams mit Anzeigenzählungen, Countdown-Timern für Anzeigen, Durchklickungsanzeigen und Begleitanzeigen zu verbessern.

    Ihre Anzeigeninformationen werden mithilfe einer Anzeigenkonfiguration angegeben, die umfangreiche Antworten abruft, um zu ermitteln, welche Anzeigen geschaltet werden sollen. Die VAST Datei enthält auch Informationen über Begleitanzeigen und Beacon-URLs.

    Wenn Sie mit SLS nicht vertraut sind, lesen Sie die Übersicht: Spieler-Erweiterungen für Live SSAI-Dokument .

    Workflow

    Gehen Sie folgendermaßen vor, um einen Livestream mit SSAI- und Spielerverbesserungen zu erstellen:

    1. Bitten Sie Ihren Kundenbetreuer, Ihr Konto für Live-Streams, Server-Side Ad Insertion (SSAI) und die Player Enhancements for Live SSAI zu aktivieren.
    2. Erstellen einer Anzeigenkonfiguration
    3. Optional: Erstellen Sie Slate-Assets, um ungenutzte Werbezeit zu füllen
    4. Erstellen Sie einen Live-Job
    5. Erstellen Sie ein Remote-Video
    6. Konfigurieren Sie den Player
    7. Beginne zu streamen
    8. Lösen Sie Anzeigen mit Cue-Points aus (dies kann vor oder während des Live-Events erfolgen)
    9. Aufräumen

    Erstellen einer Anzeigenkonfiguration

    Eine Anzeigenkonfiguration definiert die Informationen, die erforderlich sind, um serverseitige Anzeigen in einen Live-Stream einzubeziehen. Dies umfasst die URL des Anzeigenanbieters, die erwartete Antwortart der Anzeige und ob Beacons gesendet werden sollen.

    Für SLS MÜSSEN Anzeigenkonfigurationen einen ad_configuration_expected_response_type Wert von haben Vast.

    Um eine neue Anzeigenkonfiguration zu erstellen, senden Sie eine POST Anfrage wie folgt:

    -Methode POST
    URL https://api.bcovlive.io/v1/ssai/applications
    Überschrift X-API-KEY: your API KEY
    Content-Type: Anwendung/JSON

    Alle Anfragen werden mit einem API-Schlüssel authentifiziert, der Ihnen bei der Einrichtung Ihres Kontos zur Verfügung gestellt wird. Der Schlüssel wird in der X-API-KEY Kopfzeile übergeben.

    Einzelheiten zu den Feldern des Anforderungskörpers finden Sie im Live API-Referenzdokument .

    Mit Player Enhancements for Live SSAI können die hervorgehobenen Felder innerhalb des application_ad_configuration Objekts festgelegt werden.

    Muster-Anfrage Body:

    {
  "application_ad_configuration": {
	"ad_configuration_description": "your ad config description",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "your ad server",
    "ad_configuration_client_sdk_enabled": true,
    "client_options": {
      "show_ad_break_remaining_time": true / false,
      "show_ad_remaining_time": true / false,
      "show_number_of_remaining_ads": true / false,
      "client_only_tracking": true / false
    }
  },
  "application_description": "your aaplication description"
}

    Alle client_options sind optional. client_only_tracking Hat nur den Standardwert von false.

    Erstellen Sie einen Live-Job

    Um einen Live-Job zu erstellen, senden Sie eine POST Anfrage wie folgt:

    -Methode POST
    URL https://api.bcovlive.io/v1/jobs
    Header X-API-KEY: your API KEY
    Content-Type: Anwendung/JSON

    Einzelheiten zu den Feldern des Anforderungskörpers finden Sie im Live API-Referenzdokument .

    Für einen Live-Job mit aktivierter SSAI, legen Sie ad_insertion: true in Ihrem HTTP-Anfragetext fest.

    Wenn Sie einen VOD-Clip aus dem Live-Stream erstellen möchten, schließen Sie den videocloud Parameter in den Anfragetext ein. Einzelheiten finden Sie in der Live-API: Erstellen eines VOD-Clips-Dokuments .

    Das playback_url oder playback_url_dvr Das Ihrer Anzeigenkonfigurations-ID zugeordnete Feld wird im nächsten Schritt zum Erstellen eines Remote-Video-Assets verwendet.

    Erstellen Sie ein Remote-Video

    In Video Cloud erstellen Sie ein Remote-Video, das auf Ihren Live-Stream verweist. Für diesen Schritt benötigen Sie das playback_url oder playback_url_dvr aus dem vorherigen Schritt.

    Es gibt zwei Möglichkeiten, ein Remote-Video zu erstellen:

    • Verwenden von Video Cloud Studio
    • Verwenden Sie die CMS-API

    Konfigurieren Sie den Player

    Das Abspielen Ihres Live-Streams über die Playback-API unterscheidet sich geringfügig von der Art und Weise, wie Sie ein normales Video Cloud-Video abspielen. Zusätzlich zur Angabe der Video-ID (Referenz-ID) müssen Sie ein Wiedergabe-Token angeben, das die Anzeigenkonfiguration angibt, die Sie verwenden möchten. Sie müssen auch angeben, ob Sie eine DVR-Wiedergabe wünschen.

    Erstellen Sie das Wiedergabe-Token

    Um das Wiedergabe-Token zu erstellen, senden Sie eine POST Anfrage wie folgt:

    -Methode POST
    URL https://api.bcovlive.io/v1/jobs/your live job id/playback-token
    Header X-API-KEY: your API KEY
    Content-Type: Anwendung/JSON

    Muster-Anfrage Body:

    {
   "dvr":true / false,
   "application_id":"your ad config id"
}

    Beispiel Antwort:

    {
 "playback_token": "live.JhQabcdefghijkl000000000abcdefghi"
}

    Der Antworttext enthält ein generiertes Wiedergabe-Token. Verwenden Sie dies bei einem Playback-API-Aufruf mit den folgenden Playern:

    Web-Spieler

    Einzelheiten zur Konfiguration des Brightcove Players finden Sie im Dokument Using Player Enhancements for Live SSAI with Brightcove Player .

    Native Android-Spieler

    Einzelheiten zur Konfiguration des Native Player für Android finden Sie im Dokument Using Player Enhancements for Live SSAI with the Native SDKs .

    Native iOS Spieler

    Einzelheiten zur Konfiguration des Native Player für iOS finden Sie im Dokument Using Player Enhancements for Live SSAI with the Native SDKs .

    Beginne zu streamen

    Es gibt viele Tools, um ein Live-Streaming-Event zu erstellen. Hier sind einige Encoder, mit denen Sie einen von einem CDN gelieferten Live-Stream erstellen können: (Verwenden Sie das playback_url aus einem vorherigen Schritt)

    • Telestream Wirecast Encoder
    • OBS Studio-Encoder

    Aufräumen

    Wenn dein Livestream abgeschlossen ist, kannst du die Artefakte für deinen Stream wie folgt bereinigen:

    1. Stoppen Sie Ihren Streaming-Encoder

    2. Bricht den Livejob ab.

      Der Live-Job ist nur für die Wiederverbindungszeit verfügbar, die bei der Erstellung des Auftrags angegeben wurde.

       

      Um einen Live-Job abzubrechen, senden Sie eine PUT Anfrage wie folgt:

      -Methode PUT
      URL https://api.bcovlive.io/v1/jobs/your live job id/cancel
      Header X-API-KEY: your API KEY
      Content-Type: Anwendung/JSON

      Einzelheiten finden Sie im Live API-Referenzdokument .

    3. Löschen Sie Ihre Anzeigenkonfiguration

      Um eine Anzeigenkonfiguration zu löschen, senden Sie eine DELETE Anfrage wie folgt:

      -Methode PUT
      URL https://api.bcovlive.io/v1/ssai/applications/application/your ad config id
      Header X-API-KEY: your API KEY
      Content-Type: Anwendung/JSON

      Einzelheiten finden Sie im Live API-Referenzdokument .
