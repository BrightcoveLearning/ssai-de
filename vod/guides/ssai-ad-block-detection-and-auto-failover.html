Überblick

Wenn Brightcove Player bei aktiviertem Anzeigenfailover einen im Browser vorhandenen Werbeblocker erkennt, fordert der Player automatisch einen SSAI-Stream an. Andernfalls werden IMA-Anzeigen verwendet.

Für das Failover von SSAI-Anzeigen ist Folgendes erforderlich:

  • IMA3 Plugin v3.2.0 +
  • SSAI Plugin v1.3.1 +
  • Brightcove Player Version 6.8.0+

Vorsichtsmaßnahmen

  • Die Erkennung von Werbeblockern ist sehr zuverlässig, jedoch nicht zu 100%.
  • Kunden, die diese Funktion verwenden, müssen Dynamic Delivery verwenden.
  • Es werden nur IMA- und SSAI-Plugins unterstützt (kein FreeWheel).

Einstieg

Um serverseitige Anzeigen aus Video Cloud abzuspielen, müssen Sie die folgenden Schritte ausführen:

  1. Erstellen Sie eine Anzeigenkonfiguration
  2. Erstellen Sie einen Brightcove-Player
  3. Implementieren Sie das automatische Failover mit Studio

Erstellen Sie eine Anzeigenkonfiguration

Die Anzeigenkonfiguration definiert verschiedene Aspekte der SSAI-Wiedergabe, einschließlich Anzeigenaufrufen, Beacons und anderen Konfigurationsoptionen. Ihre Anzeigenantwort kann VAST-, VMAP- oder DFP-Anzeigenregeln sein. Gehen Sie folgendermaßen vor, um eine Anzeigenkonfiguration zu erstellen:

  1. Erweitern Sie in Video Cloud Studio die Option ADMINISTRATOR Menü und wählen Serverseitige Anzeigeneinstellungen.

  2. Fügen Sie Informationen für Ihre Anzeigenkonfiguration hinzu und wählen Sie speichern.

Einzelheiten finden Sie in der Konfigurieren der serverseitigen Anzeigeneinstellungen Dokument.

Informationen zum Erstellen einer Anzeigenkonfiguration mithilfe der SSAI-API finden Sie unter Video Cloud SSAI-Anzeigenkonfigurations-API Dokument.

Erstellen Sie einen Brightcove-Player

Erstellen Sie mit Video Cloud Studio einen neuen Brightcove-Player. Einzelheiten finden Sie in der Schnellstart: Erstellen und Gestalten eines Players Dokument.

Sie müssen einen Brightcove Player Version 6.8.0+ verwenden.

Implementieren des automatischen Failovers mit Studio

Die einfachste Möglichkeit, Ihren Player für Auto-Failover-Anzeigen zu konfigurieren, ist Video Cloud Studio. Nachdem Sie eine Anzeigenkonfiguration und einen Player erstellt haben, können Sie den Player wie folgt für das automatische Failover konfigurieren:

  1. Öffne das SPIELER Modul und suchen Sie den Player, dem Sie Werbefunktionen hinzufügen möchten.
  2. Wählen Sie den Link für den Spieler, um die Eigenschaften des Spielers zu öffnen.
  3. Wählen Werbung im linken Navigationsmenü.
  4. Überprüf den Aktivieren Sie clientseitig (IMA) Kontrollkästchen.

  5. Geben Sie die URL für Ihr IMA-Anzeigen-Tag an. In diesem Beispiel verwenden wir die Beispiel-URL für Anzeigen-Tags.

    Einzelheiten zu den Werbeeigenschaften des Players finden Sie in der Konfigurieren der Player-Werbung mithilfe des Players-Moduls Dokument.

  6. Überprüf den Aktivieren Sie die serverseitige (SSAI) Kontrollkästchen.

    Die serverseitige Option wird nur angezeigt, wenn Ihr Konto für SSAI aktiviert wurde.
  7. Von dem Wählen Sie Konfiguration Wählen Sie im Dropdown-Menü die Anzeigenkonfiguration aus, die Sie diesem Player zuordnen möchten.
  8. Wenn Sie möchten, dass Overlays über Ihren Anzeigen angezeigt werden, aktivieren Sie das Kontrollkästchen Aktivieren Sie Anzeigeninformations-Overlays Kontrollkästchen. Dies beinhaltet "Weitere Informationen" und Anzeigen-Countdown-Overlays.

  9. Die Erkennung von Anzeigenblöcken mit serverseitigem Anzeigenfailover wird automatisch aktiviert, wenn Sie sowohl clientseitige als auch serverseitige Anzeigen aktivieren.

  10. Wählen speichern.

  11. Um den Player zu veröffentlichen, wählen Sie Veröffentlichen & Einbetten> Änderungen veröffentlichen.

    Das ist alles, was Sie tun müssen, um das automatische Failover für Ihren Player zu aktivieren.

Spielen Sie ein Video mit Werbung ab

Jedes Video, das Sie aus der Video Cloud abrufen und das mit Dynamic Delivery aufgenommen wurde, enthält die clientseitigen IMA-Anzeigen, wenn kein Werbeblocker erkannt wird. Wenn ein Werbeblocker erkannt wird, enthält Ihr Video serverseitige Anzeigen, die in der VMAP-Datei in Ihrer Anzeigenkonfiguration angegeben sind. Beachten Sie, dass dem Video eine Audiospur zugeordnet sein muss, damit SSAI funktioniert.
