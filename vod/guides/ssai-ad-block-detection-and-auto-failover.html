    SSAI: Ja. Adblock-Erkennung und automatisches Failover

    In diesem Thema erfahren Sie, wie Sie die Auto-Failover-Einstellung eines Brightcove Players aktivieren, um SSai-Anzeigen (Server Side Ads Insertion) anzufordern, wenn Anzeigenblocker erkannt werden.

    Übersicht

    Wenn das Anzeigen-Failover aktiviert ist und der Brightcove Player einen im Browser vorhandenen Werbeblocker erkennt, fordert der Player automatisch einen SSAI-Stream an. Andernfalls werden IMA-Anzeigen verwendet.

    Für SSAI-Anzeigen-Failover ist Folgendes erforderlich:

    • IMA3-Plugin 3.2.0+
    • SSAI-Plugin 1.3.1+
    • Brightcove Player Version 6.8.0+

    Vorbehalte

    • Die Erkennung von Werbeblockern ist sehr zuverlässig, aber nicht 100%.
    • Kunden, die diese Funktion verwenden, müssen Dynamic Delivery verwenden.
    • Es werden nur IMA- und SSAI-Plugins unterstützt (kein FreeWheel).

    Erste Schritte

    Um serverseitige Anzeigen aus Video Cloud abzuspielen, müssen Sie die folgenden Schritte ausführen:

    1. Erstellen einer Anzeigenkonfiguration
    2. Erstelle einen Brightcove Spieler
    3. Implementieren Sie Auto-Failover mit Studio

    Erstellen einer Anzeigenkonfiguration

    Die Anzeigenkonfiguration definiert verschiedene Aspekte der SSAI-Wiedergabe, einschließlich Anzeigenanrufe, Beacons und anderer Konfigurationsoptionen. Ihre Anzeigenantwort kann VAST, VMAP oder DFP Ad Rules sein. Gehen Sie folgendermaßen vor, um eine Anzeigenkonfiguration zu erstellen:

    1. Erweitern Sie in Video Cloud Studio das Menü ADMIN und wählen Sie Serverseitige Anzeigeneinstellungenaus.

    2. Fügen Sie Informationen für Ihre Anzeigenkonfiguration hinzu und wählen Sie Speichernaus.

    Weitere Informationen finden Sie im Dokument Konfigurieren von serverseitigen Anzeigeneinstellungen .

    Informationen zum Erstellen einer Anzeigenkonfiguration mit der SSAI-API finden Sie im Video Cloud SSAI Ad Config API-Dokument .

    Erstellen eines Brightcove-Players

    Erstellen Sie einen neuen Brightcove Player mit Video Cloud Studio. Einzelheiten finden Sie im Schnellstart: Erstellen und Styling eines Player-Dokuments

    Sie müssen eine Brightcove Player-Version 6.8.0+ verwenden.

    Implementierung von Auto-Failover mit Studio

    Der einfachste Weg, Ihren Player für Auto-Failover-Anzeigen zu konfigurieren, ist Video Cloud Studio. Sobald Sie eine Anzeigenkonfiguration und einen Player erstellt haben, können Sie den Player wie folgt für das automatische Failover konfigurieren:

    1. Öffnen Sie das PLAYERS-Modul und suchen Sie den Player, dem Sie Werbefunktionen hinzufügen möchten.
    2. Wählen Sie den Link für den Spieler aus, um die Eigenschaften des Spielers zu öffnen.
    3. Wählen Sie im linken Navigationsmenü Werbung aus.
    4. Aktivieren Sie das Kontrollkästchen Clientseitig aktivieren (IMA) .

    5. Geben Sie die URL für Ihr IMA-Anzeigen-Tag an. In diesem Beispiel verwenden wir die Ad Tag-Beispiel-URL.

      Einzelheiten zu den Eigenschaften der Player-Werbung finden Sie im Dokument Konfigurieren von Spielerwerbung mit dem Players Module .

    6. Aktivieren Sie das Kontrollkästchen Serverseitig (SSAI) aktivieren.

      Die serverseitige Option wird nur angezeigt, wenn Ihr Konto für SSAI aktiviert wurde.
    7. Wählen Sie im Dropdown-Menü Konfiguration auswählen die Anzeigenkonfiguration aus, die Sie diesem Player zuordnen möchten.
    8. Wenn Sie möchten, dass Overlays über Ihre Anzeigen geschaltet werden, aktivieren Sie das Kontrollkästchen Anzeigeninformations-Overlays aktivieren. Dies umfasst „Mehr erfahren“ und Überlagerungen bei der Anzeigenabzählung.

    9. Die Erkennung von Anzeigenblöcken mit serverseitigem Werbe-Failover wird automatisch aktiviert, wenn Sie sowohl clientseitige als auch serverseitige Anzeigen aktivieren.

    10. Wählen Sie Speichernaus.

    11. Um den Player zu veröffentlichen, wählen Sie Veröffentlichungs- und Einbettungsänderungen einbetten >aus.

      Das ist alles, was Sie tun müssen, um Auto-Failover für Ihren Player zu aktivieren.

    Abspielen eines Videos mit Anzeigen

    Jedes Video, das Sie aus Video Cloud abrufen und das mit Dynamic Delivery aufgenommen wurde, enthält die clientseitigen IMA-Anzeigen, wenn kein Werbeblocker erkannt wird. Wenn ein Werbeblocker erkannt wird, enthält Ihr Video serverseitige Anzeigen, die in der VMAP-Datei angegeben sind, in Ihrer Anzeigenkonfiguration. Beachten Sie, dass dem Video eine Audiospur zugeordnet sein muss, damit SSAI funktioniert.
